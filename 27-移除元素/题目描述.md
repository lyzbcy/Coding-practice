# 27. 移除元素

题目来源：https://leetcode.cn/problems/remove-element/

---

## 📝 题目描述

给你一个数组 `nums` 和一个值 `val`，你需要 **原地移除** 所有数值等于 `val` 的元素，返回移除后数组中与 `val` 不同的元素个数。

要求：

1. **必须在原数组 `nums` 上操作**，不允许使用额外数组。
2. **元素的相对顺序可以改变**。
3. 新长度之外的元素内容不重要，不会被检查。

---

## ✅ 输出规范

- 函数返回值：`k` —— 移除后数组中与 `val` 不同的元素数量。
- 数组要求：`nums` 的前 `k` 个位置存放所有不等于 `val` 的元素，顺序任意。
- `k` 之后的元素值无所谓，评测不会检查。

---

## 🔢 示例

### 示例 1

**输入：**  
`nums = [3,2,2,3]`, `val = 3`

**输出：**  
`2`, `nums = [2,2,_,_]`

**解释：**  
函数返回 `k = 2`，并且 `nums` 的前两个元素是 `2`。后续位置填什么都无所谓。

---

### 示例 2

**输入：**  
`nums = [0,1,2,2,3,0,4,2]`, `val = 2`

**输出：**  
`5`, `nums = [0,1,4,0,3,_,_,_]`

**解释：**  
返回 `k = 5`，`nums` 的前 5 个元素可以是 `0, 1, 4, 0, 3` 的任意排列。

---

## 📌 提示

- `0 <= nums.length <= 100`
- `0 <= nums[i] <= 50`
- `0 <= val <= 100`

---

## 💡 评测说明

评测机会按照以下流程验证你的代码：

```c
int[] nums = [...];
int val = ...;
int[] expectedNums = [...]; // 只包含不等于 val 的元素，且长度已知

int k = removeElement(nums, val); // 调用你的函数
assert k == expectedNums.length;
sort(nums, 0, k);                 // 对前 k 个元素排序
for (int i = 0; i < k; i++) {
    assert nums[i] == expectedNums[i];
}
```

---

**目标：** 找到一种 O(n) 时间复杂度、O(1) 额外空间的解法，且实现简单可靠。

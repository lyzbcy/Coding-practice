# 141. 环形链表

题目来源：https://leetcode.cn/problems/linked-list-cycle/

---

## 📝 题目描述

给你一个链表的头节点 `head`，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（索引从 0 开始）。注意：`pos` **不作为参数进行传递**。仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 `true`。否则，返回 `false`。

**进阶：** 你能用 `O(1)`（即，常量）内存解决此问题吗？

---

## ✅ 输出规范

- 函数返回值：`bool` 类型
  - `true`：链表中存在环
  - `false`：链表中不存在环

- 处理要求：
  1. 不能修改链表结构
  2. 不能使用额外的数据结构（如哈希表）存储节点
  3. 空间复杂度要求：O(1)

---

## 🔢 示例

### 示例 1

**输入：**  
`head = [3,2,0,-4]`, `pos = 1`

**输出：**  
`true`

**解释：**  
链表中有一个环，其尾部连接到第二个节点。

**图示：**
```
3 -> 2 -> 0 -> -4
     ^            |
     |____________|
```

---

### 示例 2

**输入：**  
`head = [1,2]`, `pos = 0`

**输出：**  
`true`

**解释：**  
链表中有一个环，其尾部连接到第一个节点。

**图示：**
```
1 -> 2
^    |
|____|
```

---

### 示例 3

**输入：**  
`head = [1]`, `pos = -1`

**输出：**  
`false`

**解释：**  
链表中没有环。

**图示：**
```
1 -> NULL
```

---

## 📌 提示

- 链表中节点的数目范围是 `[0, 10^4]`
- `-10^5 <= Node.val <= 10^5`
- `pos` 为 `-1` 或者链表中的一个 **有效索引**

---

## 💡 评测说明

评测机会按照以下流程验证你的代码：

```c
struct ListNode* head = ...;  // 构造链表（可能带环）
bool result = hasCycle(head);
// 验证 result 是否正确
```

**处理步骤：**

1. **使用快慢指针（Floyd判圈算法）**：
   - `slow` 指针每次移动一步
   - `fast` 指针每次移动两步
   - 如果链表有环，快慢指针最终会相遇

2. **判断逻辑**：
   - 如果 `fast == NULL` 或 `fast->next == NULL`，说明没有环，返回 `false`
   - 如果 `slow == fast`，说明快慢指针相遇，存在环，返回 `true`

3. **边界条件**：
   - 空链表（`head == NULL`）：返回 `false`
   - 单节点链表（`head->next == NULL`）：返回 `false`

---

**目标：** 找到一种 O(n) 时间复杂度、O(1) 额外空间的解法。

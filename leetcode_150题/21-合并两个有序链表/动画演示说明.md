# 动画演示使用说明

## 🎬 如何运行动画

1. **安装依赖**

```bash
pip install matplotlib numpy
```

2. **运行程序**

```bash
python 动画演示.py
```

3. **选择示例**

程序提供四个内置选项：

- 选项 1：示例 1（基本合并）  
  `list1 = [1,2,4]`, `list2 = [1,3,4]` → `[1,1,2,3,4,4]`

- 选项 2：示例 2（空链表）  
  `list1 = []`, `list2 = []` → `[]`

- 选项 3：示例 3（一个为空）  
  `list1 = []`, `list2 = [0]` → `[0]`

- 选项 4：自定义输入（支持任意两个升序链表）

## 📊 动画元素说明

- **蓝色链表**：list1（第一个输入链表）
- **绿色链表**：list2（第二个输入链表）
- **粉色链表**：结果链表（正在构建中）
- **黄色高亮**：当前正在比较的节点位置

---

## 🔁 动画流程

1. **初始化**：
   - 创建虚拟头节点 dummy
   - 初始化 current 指针指向 dummy

2. **双指针合并过程**：
   - 同时遍历两个链表
   - 比较当前节点值：`list1->val` 和 `list2->val`
   - 选择较小的节点，连接到结果链表
   - 移动对应链表的指针到下一个节点

3. **处理剩余节点**：
   - 当一个链表遍历完时，直接将另一个链表的剩余部分连接到结果链表
   - 因为剩余部分已经是有序的，不需要再处理

4. **结果展示**：显示最终合并后的链表

## 💡 学习建议

1. **第一次观看**：选择示例 1（[1,2,4] + [1,3,4]），观察基本的合并过程

2. **第二次**：尝试示例 3（[] + [0]），重点观察：
   - 空链表的处理
   - 如何直接返回另一个链表

3. **第三次**：输入自定义测试，例如：
   - `list1 = [1,3,5]`, `list2 = [2,4,6]`（交替合并）
   - `list1 = [1,2,3]`, `list2 = [4,5,6]`（一个先完）
   - `list1 = [4,5,6]`, `list2 = [1,2,3]`（另一个先完）

4. **最后**：结合 `教学文档.md` 理解：
   - 双指针技巧的应用
   - 虚拟头节点的优势
   - 为什么可以直接连接剩余部分

5. **深入理解**：观察每一步中：
   - 哪个节点被选择
   - 为什么选择这个节点
   - 结果链表如何逐步构建

## ❓ 常见问题

- **键盘无响应？**  
  请确保点击动画窗口使其获得焦点。如果仍无效，尝试重新运行或切换 Python 终端。

- **中文显示异常？**  
  修改脚本中的 `plt.rcParams['font.sans-serif']`，换成系统可用字体。

- **无法显示窗口？**  
  在无图形界面的环境中，可以使用 `plt.savefig` 导出每一步，但更推荐在本地运行。

- **链表太长显示不全？**  
  动画会自动调整显示范围，但如果链表过长，建议使用较短的测试用例。

- **为什么使用虚拟头节点？**  
  虚拟头节点可以简化边界处理，避免判断结果链表是否为空，统一处理第一个节点的连接。

- **为什么可以直接连接剩余部分？**  
  因为两个链表都是升序的，当一个链表遍历完时，另一个链表的剩余部分已经是有序的，可以直接连接。

- **时间复杂度为什么是 O(m+n)？**  
  需要遍历两个链表的所有节点，每个节点只访问一次，所以是 O(m+n)。

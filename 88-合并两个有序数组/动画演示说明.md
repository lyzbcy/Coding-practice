# 动画演示使用说明

## 🎬 如何运行动画演示

### 1. 安装依赖

首先确保安装了必要的 Python 库：

```bash
pip install matplotlib numpy
```

### 2. 运行程序

```bash
python 动画演示.py
```

### 3. 选择示例

程序会提示你选择示例：
- **选项 1**: 标准示例（推荐初学者）
  - `nums1 = [1,2,3,0,0,0]`, `m = 3`
  - `nums2 = [2,5,6]`, `n = 3`
  
- **选项 2**: 边界情况（nums2 为空）
  - `nums1 = [1]`, `m = 1`
  - `nums2 = []`, `n = 0`
  
- **选项 3**: 边界情况（nums1 为空）
  - `nums1 = [0]`, `m = 0`
  - `nums2 = [1]`, `n = 1`
  
- **选项 4**: 自定义输入

## 🎮 手动控制说明

动画支持**键盘手动控制**，你可以按照自己的节奏观察每一步！

### 键盘快捷键

| 按键 | 功能 |
|------|------|
| **空格键** 或 **→** | 下一步 |
| **←** | 上一步 |
| **Home** | 跳转到第一步 |
| **End** | 跳转到最后一步 |
| **Q** 或 **Esc** | 退出程序 |

### 使用技巧

1. **确保窗口焦点**: 使用键盘控制前，请先**点击窗口**，确保窗口处于焦点状态
2. **仔细观察**: 每按一次空格，仔细观察当前步骤的变化
3. **反复查看**: 可以随时按左箭头回到上一步，反复观察理解
4. **快速跳转**: 使用 Home/End 可以快速跳转到开始或结束

## 📊 动画说明

### 颜色含义

- **浅蓝色** 🟦: `nums1` 的有效元素（前 m 个）
- **浅灰色** ⬜: `nums1` 的预留空间（后 n 个位置，初始为 0）
- **浅绿色** 🟩: `nums2` 的元素
- **黄色** 🟨: 当前正在比较或填充的位置（高亮显示）

### 指针说明

- **蓝色箭头 i** ➡️: 指向 `nums1` 有效元素的当前位置
- **绿色箭头 j** ➡️: 指向 `nums2` 的当前位置
- **红色箭头 k** ➡️: 指向合并后数组的填充位置

### 动画流程

1. **初始化**: 显示初始状态，三个指针都在各自数组的末尾
2. **比较阶段**: 高亮显示正在比较的两个元素
3. **填充阶段**: 将较大的元素填充到 k 位置
4. **移动指针**: 相应的指针向前移动
5. **重复**: 按空格键继续下一步，直到所有元素都处理完毕

## 🎯 关键观察点

### 为什么从后往前？

观察动画时，特别注意：
- 当 `k` 指针在填充时，它指向的位置是**空的**（灰色区域）
- 即使 `k` 和 `i` 重叠，`i` 指向的元素已经被使用过了
- 这样就不会覆盖 `nums1` 中还未处理的元素

### 指针移动规律

- **i 和 j 同时存在时**: 比较 `nums1[i]` 和 `nums2[j]`，将较大的放到 `k` 位置
- **只有 j 存在时**: 直接将 `nums2` 的剩余元素复制到 `nums1`
- **只有 i 存在时**: 不需要移动，元素已经在正确位置

## 🔧 为什么使用手动控制？

手动控制模式有以下优势：

1. **自主节奏**: 你可以按照自己的理解速度来控制，想停多久就停多久
2. **反复观察**: 可以随时回到上一步，反复观察不理解的地方
3. **深入思考**: 每一步都有足够的时间思考"为什么这样做"
4. **学习效果更好**: 主动控制比被动观看更容易理解和记忆

## ❓ 常见问题

### Q: 动画窗口没有显示？
A: 确保安装了 matplotlib，并且系统支持图形界面。如果是在服务器上，可能需要使用 `plt.savefig()` 保存图片而不是显示。

### Q: 中文显示乱码？
A: 程序已经设置了中文字体，如果仍有问题，可以修改 `plt.rcParams['font.sans-serif']` 为你的系统支持的中文字体。

### Q: 键盘控制无效？
A: 请确保**点击窗口**使其获得焦点，然后再按键盘。如果还是无效，可能是 matplotlib 后端的问题，可以尝试重新运行程序。

### Q: 如何暂停/继续？
A: 动画默认就是暂停状态，需要手动按空格键才会前进。你可以随时停止观察，想继续时再按空格。

# 二维最接近点对问题

题目来源：算法设计与分析课程作业

---

## 📝 题目描述

给定平面上 n 个点，找出其中的一对点的距离，使得在这 n 个点的所有点对中，该距离为所有点对中最小的。

---

## ✅ 输出规范

- 输入：第一行一个整数 n，表示点的个数。接下来 n 行，每行两个整数 x, y，表示一个点的行坐标和列坐标。
- 输出：仅一行，一个实数，表示最短距离，四舍五入保留 2 位小数。

---

## 🔢 示例

### 示例 1

**输入：**  
```
4
0 0
0 1
1 0
1 1
```

**输出：**  
```
1.00
```

**解释：**  
四个点分别为 (0,0), (0,1), (1,0), (1,1)。  
最短距离为点 (0,0) 和 (0,1) 之间的距离，或 (0,0) 和 (1,0) 之间的距离，均为 1.00。

---

### 示例 2

**输入：**  
```
3
0 0
1 1
2 2
```

**输出：**  
```
1.41
```

**解释：**  
三个点共线，最短距离为相邻两点之间的距离，约为 1.41（√2）。

---

### 示例 3

**输入：**  
```
2
0 0
3 4
```

**输出：**  
```
5.00
```

**解释：**  
只有两个点，最短距离为 5.00（√(3²+4²) = 5）。

---

## 📌 提示

- `1 <= n <= 1e5`
- `0 <= x, y <= 1e9`
- 所有点的坐标都是整数
- 可能存在重复点（此时最短距离为 0）

---

## 💡 评测说明

评测机会按照以下流程验证你的代码：

1. 读取点的个数 n
2. 读取 n 个点的坐标
3. 调用你的算法计算最短距离
4. 输出结果，保留 2 位小数

**注意：**
- 结果需要四舍五入到 2 位小数
- 使用 `printf("%.2f", result)` 可以自动四舍五入
- 时间复杂度要求：O(n log² n) 或更优

---

## 🎯 算法提示

本题可以使用以下方法：

1. **暴力枚举**：O(n²)，适用于 n 较小的情况
2. **分治算法**：O(n log² n) 或 O(n log n)，适用于大规模数据
3. **随机化算法**：期望 O(n log n)

推荐使用分治算法，这是经典的算法设计问题。

---

**目标：** 找到一种高效的分治算法，在 O(n log n) 或 O(n log² n) 时间内求解最近点对问题。
